# 📁 1-setup/ - 초기 설정 및 구성

이 폴더는 Nexus 템플릿의 초기 설정과 구성을 담당하는 스크립트들을 포함합니다.

## 📌 주요 기능
- AWS 서비스 설정 구성
- 환경 변수 생성 및 관리
- 프로젝트 초기화

## 📄 포함된 파일

### configure.sh
- **목적**: 대화형 설정 마법사
- **기능**: 
  - 사용자로부터 서비스 이름, AWS 리전 등 입력받기
  - .env.deploy 파일 생성
  - AWS 계정 ID 자동 감지
- **사용 시기**: 처음 템플릿을 설정할 때

### setup.sh
- **목적**: 빠른 설정 (기본값 사용)
- **기능**:
  - 기본값으로 자동 설정
  - 타임스탬프 기반 고유 서비스명 생성
  - 최소한의 사용자 입력으로 빠른 시작
- **사용 시기**: 빠른 테스트나 데모가 필요할 때

## 🔧 사용 방법

```bash
# 대화형 설정 (권장)
bash 1-setup/configure.sh

# 빠른 설정 (기본값)
bash 1-setup/setup.sh
```

## 📝 생성되는 파일
- `.env.deploy` - 전체 프로젝트 배포 설정
- `frontend/.env` - 프론트엔드 환경 변수
- `backend/.env` - 백엔드 환경 변수

## ⚠️ 주의사항
- AWS CLI가 미리 구성되어 있어야 함 (`aws configure`)
- 생성된 .env 파일들은 절대 Git에 커밋하지 않음
- 서비스명은 AWS 리소스 네이밍 규칙을 따라야 함 (소문자, 하이픈만 사용)
